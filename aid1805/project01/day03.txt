项目相关内容介绍

项目开发流程
    做需求分析--->概要设计--->项目计划--->详细设计--->编码测试--->项目测试--->调试修改--->项目发布
    1.需求分析：理解用户需求，提供需求文档，和用户进行确认
        包含：功能，部署环境，应对人群，并发量，流量控制，特殊需求，可行性分析
    2.概要设计：对项目进行初步的分析和整体设计；形成概要设计文档．
        确定项目技术思路，确定使用框架，分为多少模块
    3.项目计划：按照指定项目的前驱后继开发流程，给定项目周期，设置时间节点，做好项目分工；
        作为项目负责人最重要的工作之一
        eg. project:甘特图
    4.详细设计：项目具体实现技术，项目的技术点阐述；
        编码阐述，逻辑流程，数据结构，每个模块功能等等
        eg. visio:流程图　 mindmanage:思维导图
    5.编码测试：按照预订设计编码实现；进行bug调试，进行技术公关，进行基本的代码测试．
        注释占15%-20%
        eg. pycharm:集成开发工具  sublim/vim/atom:编译器
    6.项目测试：对项目的功能进行集中测试，整理测试文档，对项目结果负责
        测试说明书
    7.调试修改：根据测试结果对项目进行优化修改
    8.项目发布：完成项目的上线部署和发布工作，编写使用说明，或者操作说明文档
        eg. 项目开发工具：github  svn

项目注意事项
    *能够按时完成项目和项目时间不足
    *小组中人员能力差距越大越容易发生冲突
    *没有按照计划完成任务

**电子词典项目**  
客户端
    1.用户注册
    2.登录　登录后才能进行其它操作
    3.单词查询
        显示：　hi  单词解释
    4.查看历史记录
        显示： 张三　　2019-5-1 12:12:00  hi
    5.退出
服务器端
    1.数据库要存储注册人员信息和历史记录
    2.要允许多个用户能够同时操作
    3.建议使用tcp完成网络传输
    4.单词使用单词本进行处理
    提示：单词本每个单词占一行
         单词和解释之间一定有空格
         单词按照顺序排列

1.客户端框架(多少模块，两层界面怎么切换)
2.服务器端框架(使用什么方式并发，多少个功能模块)
3.使用什么方式通信
4.数据表设计(几个表，每个表存什么)


技术点

网络并发tcp通信　　多进程　多线程　IO多路复用 协程
文件处理：open  文件IO技术
数据库：mysql  pymysql
数据表设计(dict)：用户　　user--->用户名(不允许重复)　密码
                历史记录　　hist--->用户名　时间　word

服务器端                  客户端

并发接收连接               发送连接请求
# 一级界面
用户注册                  发送注册请求
断开连接结束子进程          用户退出
用户登录                  登录请求
# 二级界面
查词                      发送查词请求--->不断发送要查的单词
查看历史记录               发送查看记录请求
                         用户注销，返回到上一级界面

创建数据库： 
  create database mydict default charset=utf8;
创建user表：
  1.create table user (name varchar(64) not null primary key,passwd varchar(16) default '000000');
  2.create table user(name varchar(64) not null, passwd varchar(16), primary
key (name));
创建hist表：
  create table hist (name varchar(64) not null,time varchar(128),word varchar(32) not null);
